# 测试框架说明

本文档描述项目当前测试框架的完整落地方案，从架构设计、环境配置到执行流程一应俱全，帮助新同学快速上手并保障质量门禁落地。

适用仓库：`bare-algo`（Next.js 16 + React 19 + TypeScript 5）

## 1. 架构总览

```
┌────────────────────────────────────────────────────────┐
│                  CI Pipeline (.github/workflows/ci.yml)│
│                                                        │
│   ┌─────────┐   ┌─────────────┐   ┌──────────────┐    │
│   │  lint    │──▶│ unit-tests  │   │    build     │    │
│   │         │   │ (覆盖率门禁) │   │              │    │
│   └────┬────┘   └─────────────┘   └──────┬───────┘    │
│        │                                  │            │
│        └──────────────────────────────────▶│            │
│                                   ┌───────▼──────────┐ │
│                                   │   e2e-smoke      │ │
│                                   │ (Playwright)     │ │
│                                   └──────────────────┘ │
└────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────┐
│       本地可视化报告                      │
│  test-report-visualize.mjs               │
│  → HTML / Markdown / JSON 三格式输出      │
│  → 环形评分 + 覆盖率进度条 + 文件分布图  │
└──────────────────────────────────────────┘
```

| 层级           | 框架                        | 职责                                             | 执行环境        |
| -------------- | --------------------------- | ------------------------------------------------ | --------------- |
| 单元/组件测试  | Vitest + Testing Library    | 组件渲染、交互、纯函数逻辑、构建脚本             | jsdom / Node.js |
| E2E 冒烟测试   | Playwright                  | 页面可访问性、路由可用性、关键交互闭环           | Chromium        |
| 测试报告可视化 | `test-report-visualize.mjs` | 汇总 Vitest + Playwright + Coverage 为可视化报告 | Node.js         |

## 4. 配置文件详解

### 4.1 `vitest.config.ts`

```ts
export default defineConfig({
  resolve: {
    alias: { '@': path.resolve(__dirname, 'src') }, // 路径别名 @ → src
  },
  test: {
    globals: true, // 全局注入 describe/it/expect，无需显式 import
    environment: 'jsdom', // 默认 DOM 环境
    setupFiles: ['./tests/setup/vitest.setup.ts'],
    include: ['tests/**/*.test.{ts,tsx}'],
    coverage: {
      provider: 'v8', // 使用 V8 原生覆盖率（比 Istanbul 更快）
      reporter: ['text', 'json-summary', 'html'],
      reportsDirectory: 'coverage',
      thresholds: {
        // 硬门禁：任一未达标则测试失败
        lines: 85,
        functions: 85,
        statements: 85,
        branches: 70,
      },
    },
  },
});
```

**关键设计决策：**

- **`globals: true`**：允许测试文件直接使用 `describe`、`it`、`expect`，无需逐文件 import
- **`environment: 'jsdom'`**：组件测试需要 DOM，`tests/lib/**` 和 `tests/scripts/**` 下的逻辑测试实际在 Node 环境运行
- **覆盖率阈值**：`branches` 设为 70（低于其他三项的 85），因为分支覆盖对复杂条件逻辑难度更高

> [!CAUTION]
> 任一覆盖率指标未达标，`pnpm test:unit:report` 会以非零退出码终止，CI 流水线直接阻塞。

### 4.2 `vitest.setup.ts` 全局预设

该文件在每个测试文件运行前自动执行，提供以下预设：

| 预设内容                           | 说明                                                       |
| ---------------------------------- | ---------------------------------------------------------- |
| `cleanup()`                        | 每个测试用例后自动清理 DOM，防止状态泄漏                   |
| `@testing-library/jest-dom/vitest` | 注入 `toBeInTheDocument()`、`toHaveAttribute()` 等扩展断言 |
| `window.matchMedia` mock           | jsdom 不支持 CSS 媒体查询，mock 后避免组件报错             |
| `IntersectionObserver` mock        | jsdom 不支持 Intersection Observer API                     |
| `ResizeObserver` mock              | jsdom 不支持 Resize Observer API                           |
| `next/link` mock                   | 将 Next.js `<Link>` 转为普通 `<a>` 标签，避免路由依赖      |
| `next/navigation` mock             | mock `usePathname()` 返回 `/problems`，解除导航上下文依赖  |

### 4.3 `playwright.config.ts`

```ts
export default defineConfig({
  testDir: './tests/e2e', // E2E 测试目录
  timeout: 45_000, // 单用例超时 45 秒
  fullyParallel: false, // 顺序执行（避免端口冲突）
  workers: process.env.CI ? 1 : undefined, // CI 单 worker
  outputDir: 'test-results/playwright/artifacts',

  reporter: [
    ['list'], // 终端实时输出
    ['json', { outputFile: '...' }], // JSON（供可视化脚本）
    ['html', { outputFolder: '...' }], // HTML（浏览器查看）
  ],

  use: {
    baseURL: process.env.PLAYWRIGHT_BASE_URL ?? 'http://127.0.0.1:3001',
    trace: 'retain-on-failure', // 失败时保留 trace
    screenshot: 'only-on-failure', // 失败时截图
    video: 'retain-on-failure', // 失败时录屏
  },

  webServer: process.env.PLAYWRIGHT_SKIP_WEBSERVER
    ? undefined
    : {
        command: 'pnpm dev --port 3001', // 自动启动 dev server
        url: 'http://127.0.0.1:3001',
        reuseExistingServer: !process.env.CI, // 本地复用，CI 新建
        timeout: 180_000, // dev server 启动超时 3 分钟
      },

  projects: [{ name: 'chromium', use: devices['Desktop Chrome'] }],
});
```

**关键设计决策：**

- **端口 3001**：与 `pnpm dev` 默认的 3000 分离，避免冲突
- **`reuseExistingServer`**：本地开发时复用已运行的 dev server，加快测试启动
- **`PLAYWRIGHT_SKIP_WEBSERVER`**：设置此环境变量可跳过 webServer 自动启动，用于已有服务的场景
- **失败产物策略**：trace/screenshot/video 仅在失败时保留，减少磁盘占用

## 5. pnpm 环境规范

### 5.1 版本要求

| 工具    | 最低版本 | 当前推荐                                |
| ------- | -------- | --------------------------------------- |
| Node.js | ≥ 20     | 20 LTS                                  |
| pnpm    | ≥ 9      | 10.30.3（由 `packageManager` 字段锁定） |

仓库已声明 `packageManager` 字段，建议使用 **Corepack** 自动管理 pnpm 版本：

```bash
corepack enable
corepack prepare     # 自动安装 packageManager 声明的 pnpm 版本
```

### 5.2 安装策略

```bash
# 本地开发
pnpm install

# CI / 可复现安装（锁定 lockfile，不允许修改）
pnpm install --frozen-lockfile
```

### 5.3 禁止混用包管理器

> [!WARNING]
> 本仓库**严禁**使用 `npm install` 或 `yarn install`。
>
> - 会污染 pnpm 的 `node_modules` 硬链接结构
> - 容易引发重排冲突（权限/重命名失败）
> - 导致 CI 与本地行为不一致

## 6. 命令清单

### 6.1 完整命令说明

| 命令                         | 功能                                     |   覆盖率   |     报告输出     |
| ---------------------------- | ---------------------------------------- | :--------: | :--------------: |
| `pnpm test`                  | 运行所有单元测试（别名 `test:unit`）     |     ❌     |        ❌        |
| `pnpm test:unit`             | 运行所有单元测试                         |     ❌     |        ❌        |
| `pnpm test:unit:watch`       | 监听模式运行单元测试（文件变更自动重跑） |     ❌     |        ❌        |
| `pnpm test:coverage`         | 运行单元测试并生成覆盖率报告             |     ✅     |   覆盖率 HTML    |
| `pnpm test:unit:report`      | 运行单元测试 + 覆盖率门禁 + JSON 报告    | ✅（门禁） |  JSON + 覆盖率   |
| `pnpm test:e2e:smoke`        | 运行 E2E 冒烟测试（仅 `@smoke` 标记）    |     ❌     |   JSON + HTML    |
| `pnpm test:report:visualize` | 汇总生成可视化报告                       |     -      | HTML + MD + JSON |
| `pnpm test:workflow`         | 一键串联执行完整测试工作流               | ✅（门禁） |       全部       |

### 6.2 命令底层实现

```bash
# test:unit:report 实际执行
node -e "require('fs').mkdirSync('test-results/vitest',{recursive:true})" && \
vitest run --coverage --reporter=default --reporter=json \
  --outputFile=test-results/vitest/results.json

# test:e2e:smoke 实际执行
node -e "require('fs').mkdirSync('test-results/playwright',{recursive:true})" && \
playwright test --grep @smoke

# test:workflow 串联执行
npm run test:unit:report && npm run test:e2e:smoke && npm run test:report:visualize
```

### 6.3 推荐使用场景

**日常开发（修改后快速验证）：**

```bash
pnpm test:unit:watch     # 监听模式，修改即跑
```

**提交前最小检查：**

```bash
pnpm test:unit:report    # 单元测试 + 覆盖率门禁
pnpm test:e2e:smoke      # E2E 冒烟
```

**完整测试工作流（一键）：**

```bash
pnpm test:workflow       # 单元测试 + E2E + 可视化报告
```

**需要汇报测试结果：**

```bash
pnpm test:report:visualize
# 将最新 docs/testing/reports/<timestamp>/summary.html 发送评审
```

## 7. CI 门禁流程

配置文件：`.github/workflows/ci.yml`

### 7.1 触发条件

- `push` 到 `main` 分支
- 对 `main` 分支发起 Pull Request

### 7.2 Job 依赖链路

```
lint ──┬──▶ unit-tests
       └──▶ build ──▶ e2e-smoke
```

### 7.3 各 Job 详细说明

#### Job 1：`lint` — 代码质量检查

| 步骤           | 命令                     | 说明                             |
| -------------- | ------------------------ | -------------------------------- |
| 内容一致性校验 | `pnpm validate:content`  | 验证题目元数据与实际文件的一致性 |

| Prettier       | `pnpm format:check`      | 代码格式检查                     |
| TypeScript     | `pnpm exec tsc --noEmit` | 类型检查（不生成产物）           |

#### Job 2：`unit-tests` — 单元测试（依赖 lint）

| 步骤     | 命令                    | 说明                           |
| -------- | ----------------------- | ------------------------------ |
| 运行测试 | `pnpm test:unit:report` | 运行全部单元测试，含覆盖率门禁 |

#### Job 3：`build` — 构建验证（依赖 lint）

| 步骤 | 命令         | 说明                             |
| ---- | ------------ | -------------------------------- |
| 构建 | `pnpm build` | Next.js 生产构建，验证编译无错误 |

#### Job 4：`e2e-smoke` — E2E 冒烟测试（依赖 build）

| 步骤       | 命令                                                | 说明                          |
| ---------- | --------------------------------------------------- | ----------------------------- |
| 安装浏览器 | `pnpm exec playwright install --with-deps chromium` | 安装 Chromium 及系统依赖      |
| 运行冒烟   | `pnpm test:e2e:smoke`                               | 执行 `@smoke` 标记的 E2E 用例 |

- **缓存**：pnpm store 自动缓存（`cache: 'pnpm'`）
- **Playwright workers**：CI 环境固定 1 个 worker

## 8. 新增测试规范

### 8.1 组件测试（`tests/components/*.test.tsx`）

**命名规范**：`<ComponentName>.test.tsx`

**必须覆盖的场景：**

- ✅ 正常渲染（主要文本/按钮/链接/图片）
- ✅ 用户交互（点击、键盘、输入、hover）
- ✅ 属性变化（可选参数缺省、不同 props 组合）
- ✅ 边界条件（空态、异常输入、错误分支）

**示例：**

```tsx
import { render, screen } from '@testing-library/react';
import { describe, expect, it } from 'vitest';

import { AlgoCard } from '@/components/shared/AlgoCard';

describe('AlgoCard', () => {
  it('renders title, tags, and destination link', () => {
    render(
      <AlgoCard
        id="two-sum"
        title="Two Sum"
        description="Find indices that add up to target"
        tags={['array', 'hash-table']}
        href="/problems/array/two-sum"
        difficulty="easy"
      />
    );

    expect(screen.getByRole('link', { name: /Two Sum/i })).toHaveAttribute(
      'href',
      '/problems/array/two-sum'
    );
    expect(screen.getByText('array')).toBeInTheDocument();
  });

  it('renders without optional fields', () => {
    render(
      <AlgoCard id="reverse" title="Reverse" tags={['string']} href="/problems/string/reverse" />
    );
    expect(screen.getByRole('link', { name: /Reverse/i })).toBeInTheDocument();
  });
});
```

### 8.2 逻辑测试（`tests/lib/*.test.ts`）

**命名规范**：`<module-name>.test.ts`

**必须覆盖的场景：**

- ✅ 正常路径（主流程 happy path）
- ✅ 错误路径（异常输入、空值、越界）
- ✅ 边界输入（空数组、超大数据、特殊字符）
- ✅ 历史回归点（曾修复的 bug 应有对应用例）

### 8.3 脚本测试（`tests/scripts/*.test.ts`）

**命名规范**：`<script-name>.test.ts`

**必须覆盖的场景：**

- ✅ 脚本可正常执行（不抛异常）
- ✅ 输出文件存在且非空
- ✅ 关键字段/结构正确（JSON 格式、必要键值）

### 8.4 E2E 冒烟测试（`tests/e2e/*.spec.ts`）

**编写要求：**

- 用例标题必须包含 `@smoke` 标记
- 优先覆盖关键路径（核心页面加载/渲染、高频交互）
- 不追求复杂逻辑断言，重点验证「能用」
- 失败时优先查看 trace/video/screenshot 产物排查

**示例：**

```ts
import { expect, test } from '@playwright/test';

test('@smoke home page renders', async ({ page }) => {
  const response = await page.goto('/', { waitUntil: 'domcontentloaded' });
  expect(response?.ok()).toBeTruthy();
  await expect(page.locator('main')).toBeVisible();
});
```

## 9. 覆盖率门禁

### 9.1 当前阈值

| 指标       | 阈值  | 说明                                   |
| ---------- | ----- | -------------------------------------- |
| Lines      | ≥ 85% | 行覆盖率                               |
| Functions  | ≥ 85% | 函数覆盖率                             |
| Statements | ≥ 85% | 语句覆盖率                             |
| Branches   | ≥ 70% | 分支覆盖率（门槛更低，因分支复杂度高） |

### 9.2 判定优先级

优先关注的顺序（从最易遗漏到最直观）：

1. **`branches`** — 分支覆盖最容易遗漏真实风险（`if/else`、`switch`、`??`、`&&`）
2. **`functions`** — 确保所有功能入口被测试
3. **`lines` / `statements`** — 基础行覆盖

### 9.3 查看覆盖率报告

```bash
# 终端查看概要
pnpm test:coverage

# 浏览器查看详细覆盖率
# 打开 coverage/index.html
```

# 1. 安装依赖
pnpm install

pnpm test:unit:watch

pnpm test:unit:report   # 单元测试 + 覆盖率门禁

### 场景 D：完整测试 + 可视化报告（一键）

```bash
pnpm test:workflow
```

### 场景 E：仅更新可视化报告

```bash
pnpm test:report:visualize
```

## 12. 常见问题排查

### 12.1 `ERR_CONNECTION_REFUSED`（E2E 测试）

**症状**：Playwright 无法连接到 dev server

**排查步骤**：

1. 检查 `playwright.config.ts` 的 `baseURL` 是否为 `http://127.0.0.1:3001`
2. 确认 3001 端口未被其他进程占用：
   ```powershell
   netstat -ano | findstr :3001
   ```
3. 检查 `webServer.timeout` 是否足够（默认 180 秒）
4. 尝试设置环境变量手动启动：
   ```bash
   pnpm dev --port 3001
   # 另一终端
   set PLAYWRIGHT_SKIP_WEBSERVER=1 && pnpm test:e2e:smoke
   ```

### 12.2 覆盖率有值但 CI 失败

**症状**：测试全部通过，但 `test:unit:report` 以非零码退出

**排查步骤**：

1. 检查 `coverage/coverage-summary.json` 中 `total` 字段的各项百分比
2. 重点关注 `branches` 是否低于 70%
3. 使用 `coverage/index.html` 定位未覆盖的分支

### 12.3 动态导入告警（Vite warning）

**症状**：运行测试时出现 `problem-loaders.generated.ts` 动态导入的静态分析告警

**说明**：该文件使用动态 `import()` 加载题目模块，Vite 无法静态分析所有路径。此告警**不影响测试执行**和结果正确性。

### 12.4 依赖安装冲突

**症状**：`pnpm install` 报错、权限错误、或重命名失败

**解决方案**：

```bash
# 1. 确认只使用 pnpm（不要混用 npm/yarn）
# 2. 清理 node_modules
Remove-Item -Recurse -Force node_modules
# 3. 清理 pnpm store 缓存（可选）
pnpm store prune
# 4. 重新安装
pnpm install
```

### 12.5 Playwright 浏览器缺失

**症状**：E2E 测试报错 `browserType.launch: Executable doesn't exist`

**解决方案**：

```bash
pnpm exec playwright install --with-deps chromium
```

### 12.6 测试超时

**症状**：单个用例超过 45 秒后被 Playwright 终止

**排查步骤**：

1. 检查目标页面是否需要更长加载时间（大量算法数据）
2. 确认 dev server 已正常启动并可访问
3. 如有特殊需要，可在单个测试中设置 `test.setTimeout()`

# 返回按钮智能导航机制

## 1. 需求背景

项目中有多个层级的页面，用户在各层级间跳转时需要一个"返回"按钮。核心需求：

- **有浏览历史**：返回到实际上一页（如从 `/problems/leetcode-hot-100` 进入 `/problems/array/two-sum`，返回应回到 `leetcode-hot-100`）
- **无浏览历史**（直接输入 URL / 新标签页打开）：返回到逻辑上级（如 `/problems/array/two-sum` → `/problems/array`）
- **特定页面层级锁定**：`/problems/xxx` 只能返回到 `/problems`，`/problems` 只能返回到 `/`

## 2. 技术挑战

### 2.1 `window.history.length` 不可靠

浏览器新标签页可能自带历史条目（如 `chrome://newtab`），导致 `history.length > 1` 误判为有可回退历史。

### 2.2 `document.referrer` 在 SPA 中不更新

`document.referrer` 仅在完整页面加载时设置。Next.js 的 `<Link>` 组件进行客户端导航（SPA navigation）时，`document.referrer` **不会更新**，无法用来判断上一页是什么。

### 2.3 Navigation API 兼容性不足

`window.navigation.canGoBack` 可以精确判断能否回退，但仅 Chrome 102+ / Edge 102+ 支持，Firefox 和 Safari 不支持。

## 3. 解决方案：模块级导航追踪

利用 JavaScript 模块变量的特性：

- **SPA 导航中持久存在**：模块只加载一次，后续路由切换不会重新执行模块初始化
- **全页面加载时自动重置**：刷新页面 / 直接输入 URL 时，模块重新加载，变量归零

### 3.1 架构总览

```
┌─────────────────────────────────────────────────────┐
│                   Root Layout                        │
│                                                      │
│  ┌──────────────┐    ┌─────────────────────────┐    │
│  │ ClientNavbar │    │  NavigationTracker       │    │
│  └──────────────┘    │  (监听 pathname 变化)     │    │
│                      │  首次挂载跳过，后续递增    │    │
│                      └──────────┬──────────────┘    │
│                                 │                    │
│                      ┌──────────▼──────────────┐    │
│                      │   nav-tracker.ts         │    │
│                      │   navCount (模块级变量)   │    │
│                      └──────────▲──────────────┘    │
│                                 │                    │
│  ┌──────────────────────────────┴───────────────┐   │
│  │              BackButton                       │   │
│  │  hasAppNavHistory() → back() / push()         │   │
│  └───────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### 3.2 核心模块：`src/lib/nav-tracker.ts`

```typescript
let navCount = 0;

export function incrementNavCount() {
  navCount++;
}
export function hasAppNavHistory() {
  return navCount > 0;
}
```

- 模块级变量 `navCount`，SPA 导航中累加，全页面加载时自动归零
- `hasAppNavHistory()` 返回是否有过至少一次应用内导航

### 3.3 追踪组件：`src/components/shared/NavigationTracker.tsx`

```typescript
export function NavigationTracker() {
  const pathname = usePathname();
  const isFirstMount = useRef(true);

  useEffect(() => {
    if (isFirstMount.current) {
      isFirstMount.current = false; // 首次挂载 = 全页面加载，不计数
      return;
    }
    incrementNavCount(); // 后续路径变化 = SPA 导航，递增
  }, [pathname]);

  return null;
}
```

- 放在 `src/app/layout.tsx` 的 root layout 中，全局生效
- 使用 `useRef` 跳过首次挂载（全页面加载），仅在后续 SPA 导航时递增计数

### 3.4 返回按钮：`src/components/shared/BackButton.tsx`

```typescript
export function BackButton({ fallbackHref, strict = false }) {
  const router = useRouter();

  const handleBack = () => {
    if (strict) {
      router.push(fallbackHref); // 严格模式：始终跳转指定路径
    } else if (hasAppNavHistory()) {
      router.back(); // 有应用内导航历史：返回上一页
    } else {
      router.push(fallbackHref); // 无历史：跳转回退路径
    }
  };
}
```

#### Props

| 属性           | 类型      | 默认值  | 说明                                |
| -------------- | --------- | ------- | ----------------------------------- |
| `fallbackHref` | `string`  | -       | 无导航历史时的回退路径              |
| `strict`       | `boolean` | `false` | 严格模式：始终导航到 `fallbackHref` |
| `className`    | `string`  | -       | 自定义样式，覆盖默认样式            |

## 4. 两种模式使用场景

### 4.1 非严格模式（默认）

适用于**算法题页面**和 **demo 页面**，用户可能从不同入口进入：

```tsx
// VisualizerHeader.tsx - 算法题页面
<BackButton fallbackHref={`/problems/${category}`} />

// demo 页面
<BackButton fallbackHref="/problems/graph" />
```

行为：

- 从 `/problems/leetcode-hot-100` 点击进入 → `router.back()` → 回到 `leetcode-hot-100` ✅
- 从 `/problems/array` 点击进入 → `router.back()` → 回到 `array` ✅
- 直接输入 URL 打开 → `router.push('/problems/array')` → 跳转到分类页 ✅

### 4.2 严格模式

适用于需要**锁定返回层级**的页面：

```tsx
// /problems 页面 → 只能返回首页
<BackButton fallbackHref="/" strict />

// /problems/[category] 页面 → 只能返回分类列表
<BackButton fallbackHref="/problems" strict />
```

行为：

- 无论从哪里进入，始终返回到指定路径
- 确保页面层级关系清晰，不会跳到意外页面

## 5. 场景追踪验证

| 场景                                     | navCount | 行为                 | 目标页面                     |
| ---------------------------------------- | -------- | -------------------- | ---------------------------- |
| 新标签页打开 `/problems/array/two-sum`   | 0        | `push(fallbackHref)` | `/problems/array`            |
| 从 `/problems/leetcode-hot-100` 点击进入 | ≥1       | `router.back()`      | `/problems/leetcode-hot-100` |
| 首页 → 分类 → 题目，层层点入             | ≥2       | `router.back()`      | 上一页（分类页）             |
| `/problems/array`（strict）无论从哪来    | 任意     | `push('/problems')`  | `/problems`                  |
| `/problems`（strict）无论从哪来          | 任意     | `push('/')`          | `/`（首页）                  |

## 7. 为什么不用其他方案

| 方案                   | 问题                                                   |
| ---------------------- | ------------------------------------------------------ |
| `history.length > 1`   | 新标签页可能自带历史条目，误判为有历史                 |
| `document.referrer`    | SPA 客户端导航不更新，无法检测应用内跳转               |
| `Navigation API`       | 仅 Chrome/Edge 支持，Firefox/Safari 不兼容             |
| `sessionStorage` 计数  | 同标签页刷新不会重置，可能导致误判；需手动管理生命周期 |
| 模块级变量（当前方案） | ✅ SPA 中持久、全页面加载时自动重置、无兼容性问题      |

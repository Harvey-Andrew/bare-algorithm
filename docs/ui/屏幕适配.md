# 屏幕适配改造记录

> Bare Algo 算法可视化项目的移动端 / 桌面端自适应改造完整清单。
> 设计原则：**移动优先（Mobile First），渐进增强桌面端（Desktop Enhanced）**。

---

## 1. 全局视口与触控基座

### 1.1 Viewport 配置

- **文件**: `src/app/layout.tsx`
- **改动**: 新增 `export const viewport = { width: 'device-width', initialScale: 1, viewportFit: 'cover', maximumScale: 1 }`
- **效果**: 防止 iOS 输入框聚焦时自动放大

### 1.2 100dvh 渐进增强

- **文件**: `src/app/globals.css`, `src/components/shared/VisualizerLayout.tsx`
- **改动**: 所有 `100vh` 改为双重声明 `100vh` + `100dvh`
- **效果**: 解决 iOS Safari 底部栏伸缩导致布局跳动

### 1.3 scrollbar-gutter 修复

- **文件**: `src/app/globals.css`
- **改动**: `scrollbar-gutter: stable` 仅在 `@media (hover: hover)` 下生效
- **效果**: 防止移动端悬浮滚动条产生多余边距

### 1.4 背景渐变优化

- **文件**: `src/app/globals.css`
- **改动**: `body::before` 移动端简化为纯色背景 `oklch(0.11 0.01 265)`，仅 `min-width: 1024px` 启用 fixed 渐变 + GPU 加速
- **效果**: 规避 iOS Safari 因 `fixed` + `will-change` 引发的滚动闪烁

### 1.5 全局触控优化

- **文件**: `src/app/globals.css`
- **改动**: 全局添加 `touch-action: manipulation` 和 `-webkit-tap-highlight-color: transparent`
- **效果**: 移除 300ms 点击延迟和触控高亮框

### 1.6 安全区适配

- **文件**: `src/app/globals.css`
- **改动**: `body` 添加 `padding-top/left/right: env(safe-area-inset-*)`
- **效果**: 适配刘海屏等异形屏设备

## 3. 可视化器页面改造（核心）

> 这是整个项目适配难度最高、收益也最大的模块。核心思路是抛弃传统 Tab 切换，引入"抽屉 + 悬浮条"的移动端方案。

### 3.1 VisualizerHeader 字体与触控适配

- **文件**: `src/components/shared/VisualizerHeader.tsx`
- **改动**:
  - 标题字号 → `text-[12px] sm:text-lg lg:text-2xl`
  - ExternalLink 图标 → `w-3 h-3 sm:w-4 sm:h-4`
  - 题解按钮文字 → `text-[10px] sm:text-base`，图标 → `w-3.5 h-3.5 sm:w-5 sm:h-5`
  - 移除所有 `target="_blank"`，链接在当前页面打开

### 3.2 执行行悬浮条 + 横屏双栏（替代 Tab 切换）

- **新建**: `src/lib/hooks/useOrientation.ts`
  - 检测三种状态：`portrait`（竖屏移动端）/ `landscape`（横屏移动端）/ `desktop`
- **新建**: `src/components/shared/FloatingCodeBar.tsx`
  - 竖屏底部悬浮条：显示 `<> L{行号} {消息}`
  - 点击展开底部抽屉（70vh），展示完整 CodePanel
  - 抽屉带 slideUp 动画
- **重写**: `src/components/shared/VisualizerLayout.tsx`
  - **去掉 Tab 切换**（`activeTab` state 完全移除）
  - 三种布局模式：
    | 模式 | 条件 | 布局 |
    | ---------- | --------------- | ------------------------ |
    | 竖屏移动端 | `portrait` | 可视化全屏 + 底部悬浮条 |
    | 横屏移动端 | `landscape` | 7:5 双栏（grid-cols-12） |
    | 桌面端 | `desktop (lg+)` | 7:5 双栏（不变） |
  - 输入控件改为跟在题解后面同行，不再单独占一行
- **修改**: `src/components/visualizer/GenericVisualizer.tsx`
  - 向 VisualizerLayout 传递 `highlightLine={currentFrame.line}` 和 `message={currentFrame.message}`
  - 移动端隐藏 LOG 区域（`hidden lg:flex`），信息已在悬浮条展示

### 3.3 播放控制栏缩小

- **文件**: `src/components/shared/UnifiedPlayerControls.tsx`
- **改动**:
  - 外层 padding → `p-1 sm:p-2`，gap → `gap-0.5 sm:gap-2`
  - 按钮触摸区 → `min-h-[36px] min-w-[36px] sm:min-h-[44px] sm:min-w-[44px]`
  - 播放按钮 → `w-8 h-8`
  - 进度文字 → `text-[10px] sm:text-xs`

### 3.4 返回按钮缩小

- **文件**: `src/components/shared/BackButton.tsx`
- **改动**: 移动端 padding → `p-1`，图标 → `w-3.5 h-3.5`，去掉边框

### 3.5 方法切换居中

- **文件**: `src/components/shared/VisualizerLayout.tsx`
- **改动**: 移动端方法标签切换 `flex justify-center sm:justify-start`

### 3.6 通用可视化缩放

- **新建**: `src/components/shared/ScaleToFit.tsx`
  - 使用 `ResizeObserver` + `MutationObserver` 自动缩放内容
  - 仅缩小不放大，带 0.15s 过渡动画
- **修改**: `src/components/visualizer/GenericVisualizer.tsx`
  - 所有可视化内容包裹 `<ScaleToFit>` 组件

### 3.7 代码面板字号自适应

- **文件**: `src/components/shared/CodePanel.tsx`
- **改动**: 默认字号通过 `useState` 惰性初始化，移动端（≤640px）默认 12px，桌面端 16px

## 11. PC 端不变性保证

所有改动均采用 Tailwind 响应式前缀约束：

- 移动端样式为默认值（无前缀或 `max-*`）
- 桌面端样式通过 `sm:`/`lg:` 显式保留原规则
- 可视化双栏 7:5 比例、组件位置、搜索交互在 `lg+` 下完全不变

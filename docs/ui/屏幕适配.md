# 屏幕适配改造记录

> Bare Algo 算法可视化项目的移动端 / 桌面端自适应改造完整清单。
> 设计原则：**移动优先（Mobile First），渐进增强桌面端（Desktop Enhanced）**。

---

## 1. 全局视口与触控基座

### 1.1 Viewport 配置

- **文件**: `src/app/layout.tsx`
- **改动**: 导出 `export const viewport: Viewport = { width: 'device-width', initialScale: 1, maximumScale: 1, viewportFit: 'cover' }`
- **效果**: 防止 iOS 输入框聚焦时自动放大

### 1.2 100dvh 渐进增强

- **文件**: `src/app/globals.css`、`src/app/layout.tsx`
- **改动**:
  - `globals.css` 中 `body` 同时声明 `min-height: 100vh` 和 `min-height: 100dvh`
  - `layout.tsx` 中 `<body>` 通过 inline style 设置 `minHeight: '100dvh'`
  - `VisualizerLayout` 容器高度也使用 `h-[calc(100vh-4rem)]` + inline `style={{ height: 'calc(100dvh - 4rem)' }}` 双重声明
- **效果**: 解决 iOS Safari 底部栏伸缩导致布局跳动

### 1.3 scrollbar-gutter 修复

- **文件**: `src/app/globals.css`
- **改动**: `scrollbar-gutter: stable` 仅在 `@media (hover: hover)` 下对 `html` 生效
- **效果**: 防止移动端悬浮滚动条产生多余边距

### 1.4 背景渐变优化

- **文件**: `src/app/globals.css`
- **改动**: `body::before` 默认使用纯色背景 `oklch(0.11 0.01 265)`，仅在 `@media (min-width: 1024px)` 下启用 `fixed` 多段渐变 + `will-change: transform` GPU 加速
- **效果**: 规避 iOS Safari 因 `fixed` + `will-change` 引发的滚动闪烁

### 1.5 全局触控优化

- **文件**: `src/app/globals.css`
- **改动**: `html` 全局添加 `touch-action: manipulation` 和 `-webkit-tap-highlight-color: transparent`
- **效果**: 移除 300ms 点击延迟和触控高亮框

### 1.6 安全区适配

- **文件**: `src/app/globals.css`
- **改动**: `body` 添加 `padding-top/left/right: env(safe-area-inset-*)`
- **效果**: 适配刘海屏等异形屏设备

### 1.7 全局滚动条样式

- **文件**: `src/app/globals.css`
- **改动**: 全局设置 `scrollbar-width: thin; scrollbar-color: #475569 #1e293b;` 以及 WebKit 伪元素样式（8px 宽、深色圆角滑块）
- **效果**: 统一深色主题下的滚动条外观

## 3. 可视化器页面改造（核心）

> 这是整个项目适配难度最高、收益也最大的模块。核心思路是抛弃传统 Tab 切换，引入"抽屉 + 悬浮条"的移动端方案。

### 3.1 VisualizerHeader 字体与触控适配

- **文件**: `src/components/shared/VisualizerHeader.tsx`
- **改动**:
  - 三按钮（题解/输入/随机）共用基础样式：`px-2 sm:px-3 py-1 text-xs sm:text-base`
  - 题目标题通过 `ProblemTitle` 组件渲染，带 `ExternalProblemLink`（力扣外链图标 `h-3 w-3 sm:h-3.5 sm:w-3.5`）
  - 题解按钮 `SolutionButton` → `text-xs sm:text-base`，BookOpen 图标 → `w-3.5 h-3.5 sm:w-5 sm:h-5`
  - `InputControl` 移动端使用弹窗模式输入，桌面端使用内联 input（通过 `useIsDesktop` Hook 判断）
  - `LegendDisplay` 图例区域在移动端隐藏（`hidden sm:flex`），桌面端占剩余空间

### 3.2 执行行悬浮条 + 横屏双栏（替代 Tab 切换）

- **新建**: `src/lib/hooks/useOrientation.ts`
  - 基于 `matchMedia` 检测三种状态：
    - `desktop`：屏幕宽度 ≥ 1024px（lg 断点）
    - `landscape`：宽度 < 1024px 且横屏
    - `portrait`：宽度 < 1024px 且竖屏
    - `null`：SSR / hydration 前（用于 SSR fallback 纯 CSS 响应式）
- **新建**: `src/components/shared/FloatingCodeBar.tsx`
  - 竖屏底部悬浮条：显示 `<Code> L{行号} {消息}`
  - 点击展开底部抽屉（`h-[70vh]`），展示完整 CodePanel
  - 抽屉带 `slideUp 0.25s ease-out` 动画
  - 通过 `FloatingCodeBarContext` 向 CodePanel 注入 `highlightLine` 和 `onClose` 回调
- **重写**: `src/components/shared/VisualizerLayout.tsx`
  - **去掉 Tab 切换**（`activeTab` state 完全移除）
  - 四种渲染路径：

    | 路径         | 条件                   | 布局                                                                  |
    | ------------ | ---------------------- | --------------------------------------------------------------------- |
    | SSR fallback | `orientation === null` | 纯 CSS：`lg:hidden` 显示可视化+悬浮条，`hidden lg:grid` 显示 7:5 双栏 |
    | 竖屏移动端   | `portrait`             | 可视化全屏 + 底部 FloatingCodeBar                                     |
    | 横屏移动端   | `landscape`            | 7:5 双栏（`grid-cols-12`，`gap-2`）                                   |
    | 桌面端       | `desktop`              | 7:5 双栏（`grid-cols-12`，`gap-4`）                                   |

  - Header 采用 `flex-wrap sm:flex-nowrap` 布局，方法切换标签在移动端 `w-full order-last`

### 3.3 GenericVisualizer 适配

- **文件**: `src/components/visualizer/GenericVisualizer.tsx`
- **改动**:
  - 向 `VisualizerLayout` 传递 `highlightLine={currentFrame.line}` 和 `message={currentFrame.message}`
  - 移动端隐藏 LOG 区域（`hidden lg:flex`），信息已在悬浮条展示
  - 可视化内容区域使用 `p-2 sm:p-4 lg:p-0` 差异化内边距

### 3.4 播放控制栏缩小

- **文件**: `src/components/shared/UnifiedPlayerControls.tsx`
- **改动**:
  - 外层 → `gap-1 sm:gap-2 p-1.5 sm:p-2`
  - 控制按钮触摸区 → `min-h-[40px] min-w-[40px] sm:min-h-[44px] sm:min-w-[44px]`
  - 播放按钮 → `w-9 h-9 sm:w-8 sm:h-8`（移动端稍大方便触控）
  - 进度文字 → `text-xs font-mono`，宽度 `w-10 sm:w-12`
  - 速度选项 `0.5x` 和 `4x` 在移动端隐藏（`hidden sm:inline-flex`）
  - 快捷键提示仅在 `lg` 以上显示（`hidden lg:block`）

### 3.5 返回按钮适配

- **文件**: `src/components/shared/BackButton.tsx`
- **改动**:
  - 默认样式为 `hidden sm:inline-flex p-2.5` + 圆角边框，移动端默认隐藏
  - 图标 → `w-3.5 h-3.5 sm:w-5 sm:h-5`
  - 支持通过 `className` prop 覆盖默认样式（如分类页自定义按钮外观）
  - 内置智能导航逻辑：`strict` 模式使用 `router.push`，非 strict 模式优先使用 `history.back()` 保留滚动位置

### 3.6 方法切换排布

- **文件**: `src/components/shared/VisualizerLayout.tsx`
- **改动**: 移动端方法标签切换 `w-full order-last`，桌面端 `sm:w-auto sm:order-0`

### 3.7 通用可视化缩放

- **新建**: `src/components/shared/ScaleToFit.tsx`
  - 使用 `ResizeObserver` + `MutationObserver` 自动缩放内容
  - 仅缩小不放大（`Math.min(scaleX, scaleY, 1)`），带 `0.15s ease-out` 过渡动画
  - `transformOrigin: 'center center'` 保持居中
- **修改**: `src/components/visualizer/GenericVisualizer.tsx`
  - 所有可视化内容包裹 `<ScaleToFit>` 组件

### 3.8 代码面板字号自适应

- **文件**: `src/components/shared/CodePanel.tsx`
- **改动**:
  - 字号范围 `[10, 12, 14, 16, 18, 20, 22, 24]px`，用户选择通过 `localStorage` 持久化
  - 默认：移动端（≤ 640px）`12px`，桌面端 `16px`
  - 当处于 FloatingCodeBar 抽屉内时，header 显示当前行号（`Line N`）和关闭按钮

## 10. PC 端不变性保证

所有改动均采用 Tailwind 响应式前缀和 CSS `@media` 查询约束：

- 移动端样式为默认值（无前缀）
- 桌面端样式通过 `sm:` / `lg:` 显式保留原规则
- 可视化双栏 7:5 比例（`col-span-7` + `col-span-5`）在横屏移动端和桌面端下完全不变
- 首页采用纯 CSS `clamp()` + 媒体查询方案，独立于 Tailwind 断点体系

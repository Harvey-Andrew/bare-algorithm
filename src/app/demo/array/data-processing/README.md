# 列表/表格数据处理 Demo

## 一、业务背景与目标

### 行业场景

**电商后台商品管理系统** - B 端运营人员日常使用的核心工具

### 角色与行为链路

1. **运营人员** 进入商品管理页面
2. 通过筛选条件（类目/状态/价格/关键词）缩小范围
3. 点击表头按字段排序（销量/价格/库存）
4. 翻页浏览具体商品
5. 查看聚合统计（总销量、类目分布、数据质量警告）

### 触发时机

- 页面加载：一次性拉取全量商品数据（或分页接口）
- 筛选/排序：前端实时计算，无需请求后端
- 翻页：前端切片渲染

## 二、数据协议

### 接口 JSON 示例

```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "id": "PROD00000001",
        "sku": "SKU00000001",
        "name": "手机数码商品1",
        "category": "手机数码",
        "price": 99900,
        "stock": 150,
        "sales": 2340,
        "createdAt": 1703500800000,
        "status": "active",
        "tags": ["热销", "促销"]
      }
    ],
    "total": 10000
  }
}
```

### 字段说明

| 字段      | 类型     | 说明                    |
| --------- | -------- | ----------------------- |
| id        | string   | 商品唯一 ID             |
| sku       | string   | SKU 编码                |
| name      | string   | 商品名称                |
| category  | string   | 类目（8 种）            |
| price     | number   | 价格（分）              |
| stock     | number   | 库存                    |
| sales     | number   | 销量                    |
| createdAt | number   | 上架时间戳              |
| status    | enum     | active/inactive/deleted |
| tags      | string[] | 标签（可选）            |

### 脏数据类型

1. **缺失字段**: ~10% 商品无 `tags` 字段
2. **重复数据**: ~3% SKU 重复
3. **异常值**:
   - ~2% 价格 ≤ 0（负数或零）
   - ~1% 库存 < 0（负数）

## 三、约束条件

| 维度         | 约束                                         |
| ------------ | -------------------------------------------- |
| 数据量级     | 10,000+ 条                                   |
| 主线程预算   | 单次计算 < 50ms                              |
| 性能警告阈值 | 排序/筛选 > 16ms                             |
| 内存上限     | ~4MB（每条约 400 bytes）                     |
| 增量更新     | 筛选条件变化时仅重算结果，不重新生成原始数据 |

---

## 四、示例输入输出

### 示例 1：正常筛选

**输入**:

- 筛选: `{ category: "手机数码", status: "active" }`
- 排序: `{ field: "sales", order: "desc" }`
- 分页: `{ page: 1, pageSize: 20 }`

**输出**:

- 符合条件的前 20 条商品
- 聚合统计: 类目下商品数、总销量
- 处理时间: < 10ms

### 示例 2：脏数据场景

**输入**: 包含重复 SKU、缺失 tags 的原始数据

**输出**:

- 正常展示所有数据
- 数据质量警告:
  - "发现 300 条重复 SKU"
  - "发现 1000 条记录缺失 tags 字段"
  - "发现 200 条记录价格异常（≤0）"

## 七、既有解决方案对比

| 方案                | 适用场景  | 复杂度     | 优点               | 缺点           |
| ------------------- | --------- | ---------- | ------------------ | -------------- |
| 前端全量计算        | < 10 万条 | O(n log n) | 响应快、无网络延迟 | 内存占用       |
| 后端分页 + 筛选     | > 10 万条 | O(1) 前端  | 内存小             | 每次操作需请求 |
| 虚拟滚动 + 前端计算 | 任意规模  | O(n) 计算  | 平衡性能与体验     | 实现复杂       |

**推荐方案**: 本 Demo 采用「前端全量计算」，适合万级数据的后台管理场景。

# 股票交易信号 Demo

## 一、业务背景与目标

### 行业场景

**金融交易终端** - 量化分析师和个人投资者日常使用的分析工具

### 角色与行为链路

1. **分析师** 选择股票和时间范围
2. 系统自动分析历史价格走势
3. 计算单次交易的最大利润（LeetCode #121）
4. 生成基于 MA 的交易信号
5. 用户根据信号做出投资决策

### 触发时机

- 页面加载：拉取历史行情数据
- 切换时间范围：重新计算指标
- 实时行情：WebSocket 推送更新（扩展）

## 六、核心算法说明

### 最大利润算法（O(n)）

```typescript
function calculateMaxProfit(prices: number[]): number {
  let minPrice = Infinity;
  let maxProfit = 0;

  for (const price of prices) {
    if (price < minPrice) {
      minPrice = price; // 更新最小买入价
    } else {
      const profit = price - minPrice;
      maxProfit = Math.max(maxProfit, profit); // 更新最大利润
    }
  }

  return maxProfit;
}
```

### 关键思路

1. **跟踪最小买入价**: 遍历过程中记录到目前为止的最低价格
2. **计算当前利润**: 假设今天卖出，利润 = 当前价 - 最小买入价
3. **更新最大利润**: 取所有可能利润的最大值

## 七、既有解决方案对比

| 方案       | 适用场景 | 复杂度     | 优点     | 缺点     |
| ---------- | -------- | ---------- | -------- | -------- |
| 暴力双循环 | 教学演示 | O(n²)      | 思路直观 | 性能差   |
| 一次遍历   | 本 Demo  | O(n)       | 最优解   | 需理解   |
| 动态规划   | 多次交易 | O(n)       | 可扩展   | 实现复杂 |
| 分治法     | 并行计算 | O(n log n) | 可并行   | 实现复杂 |

**推荐方案**: 本 Demo 采用「一次遍历」，单次交易场景的最优解。

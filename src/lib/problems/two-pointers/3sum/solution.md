你好！我是你的面试官兼技术顾问。这道题是 **LeetCode 第 15 题 "3Sum"（三数之和）**，它是数组处理与双指针算法的“圣经级”题目。

这道题考察的不仅仅是写出 $O(N^2)$ 的解法，更在于**如何优雅地去重**以及**代码的鲁棒性**。在 $N=3000$ 的数据规模下，任何 $O(N^3)$ 的暴力解法都会超时，而 $O(N^2)$ 是理论下界（因为必须遍历结果集），我们必须在 $O(N^2)$ 的基础上做到极致的常数优化。

---

## 1. 题目分析

- **复述题意**：在一个整数数组中，找出所有满足 $a + b + c = 0$ 的不重复三元组。
- **模型抽象**：**排序 + 双指针**（标准的 nSum 问题降维打击）或 **哈希表**。
- **约束分析**：
  - $3 \le N \le 3000$：这意味着 $O(N^2)$ 算法大约执行 $9 \times 10^6$ 次操作，现代 JS 引擎完全可以承受（通常 $10^7 \sim 10^8$ 为 1 秒限额）。
  - 数值范围 $-10^5 \sim 10^5$：三数之和最大 $3 \times 10^5$，远小于 JS `Number.MAX_SAFE_INTEGER`，**无需 BigInt**。
- **核心难点（边界）**：
  - **结果去重**：这是最大的坑。`[-1, 0, 1]` 和 `[0, 1, -1]` 是同一个解，必须只输出一份。
  - **元素去重**：输入可能有重复元素，如 `[0, 0, 0, 0]`，如果不处理好指针移动，会产出多组 `[0, 0, 0]`。
- **总体策略**：
  1.  **推荐解法（解法 1）**：排序 + 双指针。这是面试标准答案，利用排序后的单调性避免重复计算。
  2.  **哈希解法（解法 2）**：不依赖排序（或利用哈希判重），适用于不可变数组或特定场景，但去重逻辑复杂。
  3.  **工程优化解法（解法 3）**：基于**计数与分桶**（正数/负数/零）。当数据量大但重复元素多时，此法性能极佳。

---

## 解法 1：排序 + 双指针（标准最优解）

### 思路

这是解决 nSum 问题的通解。

1.  **排序**：首先对数组升序排序。排序不仅是为了让相同元素相邻（方便去重），更是为了利用数值的大小关系移动指针。
2.  **固定一个数**：遍历数组，固定 `nums[i]` 作为三元组的第一个数。
3.  **双指针扫描**：在 `i` 后面的区间 `[i+1, n-1]` 中，设置 `left` 和 `right` 指针，寻找 `nums[left] + nums[right] == -nums[i]`。
    - 若和 < 0：说明需要更大的数，`left++`。
    - 若和 > 0：说明需要更小的数，`right--`。
    - 若和 == 0：找到一组解，记录下来，并同时收缩 `left` 和 `right`。
4.  **去重技巧（核心）**：
    - 外层去重：若 `nums[i] == nums[i-1]`，说明该基准数已处理过，直接 `continue`。
    - 内层去重：当找到解时，若 `nums[left] == nums[left+1]`，循环 `left++`；同理处理 `right`。

### 代码实现 (LeetCode 提交版)

```ts
/**
 * 复杂度分析：
 * 时间复杂度：O(N^2) —— 数组排序为 O(N log N)，双指针遍历为 O(N^2)，总体为 O(N^2)。
 * 空间复杂度：O(log N) —— 主要是 V8 引擎排序算法的栈空间开销（不考虑输出数组占用的空间）。
 */

function threeSum(nums: number[]): number[][] {
  const n = nums.length;
  const res: number[][] = [];

  // 1. 排序：这是双指针法的前提
  // 注意：TypeScript/JS 的 sort 默认是字典序，必须传入比较函数
  nums.sort((a, b) => a - b);

  // 2. 遍历基准数 k
  for (let i = 0; i < n - 2; i++) {
    const x = nums[i];

    // 剪枝：如果当前数字已经大于0，后面不可能凑出 0（因为已排序）
    if (x > 0) break;

    // 去重 A：如果当前数字和上一个一样，跳过（防止结果集中出现重复的三元组）
    if (i > 0 && x === nums[i - 1]) continue;

    // 3. 双指针寻找另外两个数
    let left = i + 1;
    let right = n - 1;

    while (left < right) {
      const sum = x + nums[left] + nums[right];

      if (sum === 0) {
        // 找到解
        res.push([x, nums[left], nums[right]]);

        // 去重 B & C：跳过重复的 left 和 right
        while (left < right && nums[left] === nums[left + 1]) {
          left++;
        }
        while (left < right && nums[right] === nums[right - 1]) {
          right--;
        }

        // 找到解后，双指针同时收缩
        left++;
        right--;
      } else if (sum < 0) {
        // 和太小，左指针右移
        left++;
      } else {
        // 和太大，右指针左移
        right--;
      }
    }
  }

  return res;
}
```

## 解法 2：哈希集合（Set）辅助去重

### 思路

如果不进行整体排序，或者希望展示对哈希表的运用，可以使用此法。它将 3Sum 降维成 $N$ 次 2Sum。

1.  **两层循环**：外层遍历 `i`，中层遍历 `j`。
2.  **哈希查找**：计算 `target = -(nums[i] + nums[j])`，检查 `target` 是否存在。
3.  **难点处理**：
    - 由于输入未排序，我们很难像解法 1 那样通过指针跳过来去重。
    - **解决方案**：我们需要对找到的**结果三元组**本身进行排序，然后转化为字符串（如 `"-1,0,1"`）存入 `Set`，利用 Set 的唯一性去重。最后再将字符串转回数组。
4.  **注意**：为了正确性，内层循环必须小心处理“不要用到同一个元素多次”。通常做法是提前建立频率 Map，或者按顺序遍历。这里采用**双层循环 + Set** 的经典变体。

_注意：此解法在 LeetCode 上性能通常劣于解法 1，因为它涉及大量的字符串操作和哈希开销，但在面试中能体现对 Hash 的理解。_

### 代码实现 (LeetCode 提交版)

```ts
/**
 * 复杂度分析：
 * 时间复杂度：O(N^2) —— 两层循环，Set 操作平均 O(1)，但字符串序列化增加了常数开销。
 * 空间复杂度：O(N) —— 需要额外的 Set 存储中间结果和去重字符串。
 */

function threeSumHash(nums: number[]): number[][] {
  const resSet = new Set<string>(); // 用于结果去重
  const n = nums.length;

  // 为了方便后续输出有序的三元组，我们先对原数组做个浅排序
  // 虽然这里用了排序，但核心逻辑是演示 Hash 的用法。
  // 若完全不排序，则需要在存入 resSet 前对 [a,b,c] 单独排序。
  nums.sort((a, b) => a - b);

  for (let i = 0; i < n - 2; i++) {
    // 简单剪枝与去重（利用排序后的性质优化 Hash 法）
    if (nums[i] > 0) break;
    if (i > 0 && nums[i] === nums[i - 1]) continue;

    const target = -nums[i];
    const seen = new Set<number>(); // 存储当前轮次遍历过的 nums[j]

    for (let j = i + 1; j < n; j++) {
      const complement = target - nums[j];

      if (seen.has(complement)) {
        // 找到一组解: [nums[i], complement, nums[j]]
        // 由于 nums 已排序，且 j > 之前的索引，所以 nums[i] <= complement <= nums[j]
        // 这里的顺序是确定的，可以直接拼接 key
        resSet.add(`${nums[i]},${complement},${nums[j]}`);

        // 注意：这里不能 break，因为可能还有其他组合
      }

      seen.add(nums[j]);
    }
  }

  // 将去重后的字符串还原为数组
  return Array.from(resSet).map((str) => str.split(',').map(Number));
}
```

## 解法 3：计数统计与正负分桶（极致性能优化）

### 思路

当 $N$ 很大但**重复元素极多**时（例如 $N=3000$ 但只有 $-100 \sim 100$ 的数值），上述 $O(N^2)$ 实际上有很多冗余计算。我们可以根据组合数学思路优化。

1.  **分桶**：将 `nums` 拆分为三个集合：正数 `P`、负数 `N`、零 `Z`。同时用 `Map` 记录每个数字出现的次数。
2.  **分类讨论**（满足 $a+b+c=0$）：
    - **情况 1：0 + 0 + 0**：如果 0 的数量 $\ge 3$，加入 `[0, 0, 0]`。
    - **情况 2：x + 0 + (-x)**：如果 0 的数量 $> 0$，遍历正数 `p`，若对应的 `-p` 存在，加入 `[-p, 0, p]`。
    - **情况 3：x + y + z = 0**（无 0）：
      - 两个负数 + 一个正数：遍历负数对 $(n_1, n_2)$，检查 $-(n_1+n_2)$ 是否在正数集中。
      - 两个正数 + 一个负数：遍历正数对 $(p_1, p_2)$，检查 $-(p_1+p_2)$ 是否在负数集中。
3.  **优势**：我们将复杂度从 $O(N^2)$ 降到了 $O(U^2)$，其中 $U$ 是**去重后**元素的个数。在重复率高的数据下，这是最快解法。

### 代码实现 (LeetCode 提交版)

```ts
/**
 * 复杂度分析：
 * 时间复杂度：O(N + U^2) —— N 为原长度，U 为去重后元素个数。最坏情况 U=N，退化为 O(N^2)。
 * 空间复杂度：O(N) —— 存储计数 Map 和去重后的数组。
 */

function threeSumOptimized(nums: number[]): number[][] {
  const res: number[][] = [];
  const counter = new Map<number, number>();

  // 1. 计数统计
  for (const x of nums) {
    counter.set(x, (counter.get(x) || 0) + 1);
  }

  // 2. 分桶：只处理去重后的数字
  // 这里的 uniqueNums 相当于把 N 压缩到了 U
  const uniqueNums = Array.from(counter.keys()).sort((a, b) => a - b);
  const pos = uniqueNums.filter((x) => x > 0);
  const neg = uniqueNums.filter((x) => x < 0);

  // Case 1: [0, 0, 0]
  if ((counter.get(0) || 0) >= 3) {
    res.push([0, 0, 0]);
  }

  // Case 2: [x, 0, -x]
  if ((counter.get(0) || 0) > 0) {
    for (const p of pos) {
      if (counter.has(-p)) {
        res.push([-p, 0, p]);
      }
    }
  }

  // Case 3: 两个负数 + 一个正数 (neg1 + neg2 + pos = 0)
  for (let i = 0; i < neg.length; i++) {
    for (let j = i; j < neg.length; j++) {
      const n1 = neg[i];
      const n2 = neg[j];
      const target = -(n1 + n2);

      // 如果 target 是正数且存在
      if (counter.has(target)) {
        // 如果 n1 == n2，需要检查是否有至少两个 n1
        if (i === j && counter.get(n1)! < 2) continue;

        // 确保 target 在正数范围内（虽然逻辑上必然成立，但为了严谨）
        // 且避免重复：我们只找 neg + neg + pos，
        // 由于 pos > 0 且 n1, n2 < 0，所以 target 必然是正数，这组解唯一。
        res.push([n1, n2, target]);
      }
    }
  }

  // Case 4: 两个正数 + 一个负数 (pos1 + pos2 + neg = 0)
  for (let i = 0; i < pos.length; i++) {
    for (let j = i; j < pos.length; j++) {
      const p1 = pos[i];
      const p2 = pos[j];
      const target = -(p1 + p2);

      if (counter.has(target)) {
        if (i === j && counter.get(p1)! < 2) continue;
        res.push([target, p1, p2]);
      }
    }
  }

  return res;
}
```

## 解法对比与总结

| 特性           | 解法 1 (双指针)            | 解法 2 (Hash Set)     | 解法 3 (分桶计数)           |
| :------------- | :------------------------- | :-------------------- | :-------------------------- |
| **核心逻辑**   | 排序利用单调性收缩区间     | Hash 判重，字符串去重 | 组合数学，压缩数据规模      |
| **时间复杂度** | $O(N^2)$ (稳定)            | $O(N^2)$ (常数大)     | $O(N + U^2)$ (特定数据极快) |
| **空间复杂度** | $O(\log N)$                | $O(N)$                | $O(N)$                      |
| **去重难度**   | 简单 (利用 `prev != curr`) | 困难 (需序列化)       | 中等 (逻辑分类自带去重)     |
| **推荐场景**   | **通用的面试首选**         | 只有当禁止排序时考虑  | **大数据、多重复值**场景    |

### 关键测试样例

1.  **标准用例**
    - 输入: `[-1, 0, 1, 2, -1, -4]`
    - 输出: `[[-1, -1, 2], [-1, 0, 1]]`
    - 覆盖: 基本逻辑，重复元素 `[-1, -1]` 的处理。

2.  **全零用例 (边界)**
    - 输入: `[0, 0, 0, 0]`
    - 输出: `[[0, 0, 0]]`
    - 覆盖: Case 1 (零处理)，防止输出多个 `[0,0,0]`。

3.  **无解用例**
    - 输入: `[1, 2, 3]`
    - 输出: `[]`
    - 覆盖: 所有和均 $>0$，指针能否正确终止。

4.  **数量不足 (边界)**
    - 输入: `[0, 1]`
    - 输出: `[]`
    - 覆盖: 循环条件 `n-2` 或 `left < right` 的健壮性。

5.  **大量重复值 (性能)**
    - 输入: `[-2, -2, ..., 1, 1, ..., 1]` (3000个)
    - 输出: `[[-2, 1, 1]]`
    - 覆盖: 解法 3 的性能优势，解法 1 的 `skip duplicate` 逻辑是否生效。

建议面试时优先展示 **解法 1**，因为它的空间复杂度最低且不依赖额外的数据结构，最能体现对指针操作的掌控力。若面试官询问“如果数字范围很小但 N 很大怎么办？”，则抛出 **解法 3** 展现思维深度。

## 1. 题目分析

- **复述题意**：给定一个长度为 $N$ 的数组 `height`，其中 `height[i]` 代表在数轴 x 坐标 $i$ 处的一根高度为 `height[i]` 的垂直垂线。要求你找出其中任意两条垂线，使它们与 x 轴共同构成的“木桶”能装最多的水，返回这个最大水量。
- **模型抽象**：这是一个极具代表性的**双变量最优化问题**。容纳的水量取决于两个变量：两条线之间的水平距离（宽度）以及两条线中较短的那条（高度），即 $Area = \min(height[left], height[right]) \times (right - left)$。
- **约束分析**：根据 LeetCode 约束（$N \le 10^5$），时间复杂度绝不容许达到 $O(N^2)$，必须优化至 $O(N)$。这里也不涉及大树运算，标准的 $10^5 \times 10^4$ （高度最大值）不超过 `Number.MAX_SAFE_INTEGER`。
- **边界与澄清点**：
  1. $N \ge 2$，所以一定能构成一个容器，无需担心空数组或单元素。
  2. 不能倾斜容器（这意味着水面高度绝对由短板决定）。
- **总体策略**：我们将提供两种解法及其可视化：
  1. **暴力枚举法 (O(N^2))**：作为引出问题的基线解法，演示为什么这种方式存在大量“绝对不可能更好”的无效枚举计算。
  2. **双指针贪心法 (O(N))**：本题唯一的最优解，核心重点在于理解和证明“**为什么舍弃短板不仅能缩小搜索空间，且绝对不会错过最优解**”。

## 解法 1（暴力枚举法 - Brute Force）

### 思路

- **核心逻辑**：毫无思考地使用两层 `for` 循环，枚举所有可能的线对组合 `(i, j)`（其中 $i < j$），每次计算容器的水量并更新全局最大值。
- **为什么它不好**：当 $N=10^5$ 时，判断次数可达 $\approx 5 \times 10^9$ 次运算。虽然逻辑没错，但里面做了大量注定没用的工作：假如选定了 `i=0` 和 `j=10000`，且 `height[0]` 极矮，那么在保持 `i=0` 不变的情况下，往里缩 `j` 是没有意义的，因为宽度在缩小，而水面的上限被 `height[0]` 卡死，总面积只会越来越小！

### 代码实现 (不在生产环境使用)

```ts
/**
 * 复杂度分析：
 * 时间复杂度：O(N^2) —— 包含两层完整的嵌套循环穷举了 N(N-1)/2 种可能。
 * 空间复杂度：O(1) —— 除 maxArea 外没有使用额外空间。
 */

function maxAreaBruteForce(height: number[]): number {
  let maxArea = 0;
  for (let i = 0; i < height.length - 1; i++) {
    for (let j = i + 1; j < height.length; j++) {
      const area = Math.min(height[i], height[j]) * (j - i);
      maxArea = Math.max(maxArea, area);
    }
  }
  return maxArea;
}
```

## 解法 2（双指针贪心法 - Two Pointers，标答）

### 思路

- **核心逻辑**：
  我们在数组的两端各放置一个指针（`left` 和 `right`），构成了可能的最宽容器。
  此时，水的面积是由最短的一端决定的。由于我们要想探索更大的面积，我们必须向内移动指针（这会导致宽度减小），**那么我们该移动哪一边**？

  **逻辑证明：为什么移动短板？**
  假设此刻 `height[left] < height[right]`，`left` 是短板。
  如果我们移动长板 `right`（往左缩），那么新的宽度必然缩小，而新容器的高度的上限依然被原来的 `height[left]` 死死限制住，新高度 $\le height[left]$。因此：新面积 $\le height[left] \times (旧宽度 - 1)$ < 旧面积。
  也就是说，**保留短板去移动长板，面积绝对不可能增加！这是一个必败选项！**
  所以，我们唯一的出路，也是唯一的希望，是**舍弃当前的短板**，将 `left` 向右移动，期待能遇到一根足够长的板，来弥补我们因变窄而损失的宽度。

- **TS 细节**：在实现上是一个纯粹的单循环，操作极其轻量，没有任何的临时对象创建负担，常数极小。

### 代码实现 (LeetCode 提交版)

```ts
/**
 * 复杂度分析：
 * 时间复杂度：O(N) —— 左右指针不断向中心收拢，每次至少移动一个指针，直到相遇，最多执行 N 步。
 * 空间复杂度：O(1) —— 仅复用了有限的常数级变量记录指针位置及最大面积。
 */

function maxArea(height: number[]): number {
  let left = 0;
  let right = height.length - 1;
  let maxArea = 0;

  while (left < right) {
    // 根据木桶原理，高由左右两侧的短边决定
    const currentHeight = Math.min(height[left], height[right]);
    const currentWidth = right - left;
    const area = currentHeight * currentWidth;

    // 更新历史最大水量
    if (area > maxArea) {
      maxArea = area;
    }

    // “贪心收缩规则”：抛弃短的那一边（因为保留短边去收缩只会让水越来越少）
    if (height[left] < height[right]) {
      left++;
    } else {
      right--;
    }
  }

  return maxArea;
}
```

## 解法对比与总结

1. **适用场景与思考转化**：
   这个问题的本质其实是：**如何在一个 $N \times N$ 的网格（组合空间）中，通过逻辑剪枝，把 $O(N^2)$ 的搜索路径剪成唯一的 $O(N)$ 折线。**
   双指针贪心法完美的做到了这一点。每次移动短板，实际上就是在这个状态矩阵中，排除了以当前短板为边界的所有往内收缩的组合（因为它们必定比当前更差）。这种“消去整行/整列不可能解”的思想，堪称算法面试解答的高光范本。

2. **关键测试样例**：
   - **样例 1（标准功能用例，长高互博）**：
     - 输入：`height = [1, 8, 6, 2, 5, 4, 8, 3, 7]`
     - 期望输出：`49`
     - 说明：常规的 LeetCode 测试用例例图，由第 2 根柱 (h=8) 到最后一根柱 (h=7) 构成最佳容器，宽为 7，高为 7。
   - **样例 2（平坦边缘极高）**：
     - 输入：`height = [10, 1, 1, 1, 10]`
     - 期望输出：`40`
     - 说明：尽管中间很稀烂，但远端的高板保证了宽度的优势得到最大发挥。
   - **样例 3（阶梯型）**：
     - 输入：`height = [1, 2, 3, 4, 5, 6]`
     - 期望输出：`9`（`3`和`6` 或者 `4`和`5` 等能组成的面积最大值并非在远端）。
     - 测试：验证遇到持续递增的数据是否能在每次短板舍弃中找到巅峰交汇点。

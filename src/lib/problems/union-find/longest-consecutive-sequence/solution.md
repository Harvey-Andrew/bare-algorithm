## 1. 题目分析

- **复述题意**：给定一个未排序的整数数组 `nums`，要求找出其中数字连续的最长序列的长度（例如 `[1, 2, 3, 4]` 长度为 4）。注意：序列中的数字在原数组中不需要相邻。
- **模型抽象**：
  - **集合去重/哈希查找**：利用哈希表 $O(1)$ 的查找特性，寻找序列的起点并向后枚举。
  - **动态规划（哈希表状态转移）**：记录每个数字所在连续序列的长度，并在合并区间时更新边界。
  - **排序与双指针/迭代**：通过对数组排序，将空间上的离散转化为物理上的连续，再进行一次遍历。
- **约束分析**：
  - 根据通常的 LeetCode 数据规模（$N \le 10^5$），要求在 $O(N)$ 时间复杂度内完成。这意味着我们不能使用纯粹暴力的 $O(N^2)$ 枚举，标准排序 $O(N \log N)$ 在理论上不满足要求（但在工程实践中非常有效）。
  - 数值范围通常是 $[-10^9, 10^9]$。这个范围远在 JavaScript 安全整数范围（$[-2^{53}+1, 2^{53}-1]$）内，因此**不需要使用 `BigInt`**，直接使用 TS 的 `number` 类型完全安全。
- **边界与澄清点**：
  - **空数组**：当 `nums.length === 0` 时，应当返回 `0`。
  - **包含重复元素**：相同的数字不应该增加连续序列的长度（例如 `[1, 2, 2, 3]` 长度为 `3`）。
  - **包含负数和零**：负数跨越到正数的序列完全合法（如 `[-1, 0, 1]` 长度为 `3`）。
- **总体策略**：
  1.  **解法 1（哈希集合，理论与实践最优）**：利用 `Set` 寻找序列起点，符合严格的 $O(N)$ 复杂度和面试官期望的标准答案。
  2.  **解法 2（动态规划/哈希表边界拼凑法）**：适用于在线流式处理（Streaming Data），展示你对区间合并和状态转移的深刻理解。
  3.  **解法 3（原地排序迭代，工程兜底与性能反思）**：虽然理论复杂度是 $O(N \log N)$，但展示了高级 TS/Node.js 工程师对 V8 引擎底层数组内存连续性（Memory Locality）和垃圾回收（GC）开销的理解，数据量不极大时往往跑得比 Hash 更快。

## 解法 1：哈希集合寻找起点（标准 $O(N)$ 最优解）

### 思路

- **核心逻辑**：
  为了在 $O(N)$ 时间内找到最长序列，我们需要避免做无用功。如果把所有数字放入一个集合 `Set` 中，对于任何一个数字 `x`，如果集合中存在 `x - 1`，那么 `x` 肯定不是某个连续序列的**起点**；它必定属于一个更长的序列。
  因此，我们的不变量是：**只从没有左邻居（`x - 1` 不存在）的数字开始，向右枚举计算序列长度**。这样能保证数组中的每个数字最多被访问两次（一次在 `Set` 初始化，一次在 `while` 循环内部）。
- **TS 细节**：
  - 使用原生的 `Set<number>` 进行 $O(1)$ 的查找与去重。
  - V8 引擎下 `Set` 的底层实现是基于哈希表的，因此插入和查找的平均时间复杂度均为 $O(1)$。
  - 在处理循环时，维护当前最长长度 `maxLength`，并在过程中不断用 `Math.max` 刷新。

### 代码实现 (LeetCode 提交版)

```ts
/**
 * 复杂度分析：
 * 时间复杂度：O(N) —— 将元素放入 Set 需要 O(N)；遍历 Set 时，由于仅在 (num - 1) 不存在时才进入 while 循环，
 *                    这就意味着内部 while 循环只会在每个序列的起点触发。整体来看，所有序列中的元素最多被 while
 *                    遍历 1 次。总遍历次数为 N + N = 2N，即 O(N)。
 * 空间复杂度：O(N) —— 需要一个哈希集合 Set 来存储所有的数字，最坏情况下没有重复数字，大小为 N。
 */

function longestConsecutive(nums: number[]): number {
  if (nums.length === 0) return 0;

  // 使用 Set 自动去重并提供 O(1) 的查询能力
  const numSet: Set<number> = new Set(nums);
  let maxLength: number = 0;

  for (const num of numSet) {
    // 关键优化：只有当 num 是一个序列的"起点"时，才开始向后匹配
    // 如果存在 num - 1，说明 num 只是某个序列的中间节点，直接跳过
    if (!numSet.has(num - 1)) {
      let currentNum: number = num;
      let currentLength: number = 1;

      // 顺藤摸瓜，不断在 Set 中查找下一个连续数字
      while (numSet.has(currentNum + 1)) {
        currentNum += 1;
        currentLength += 1;
      }

      // 更新全局最大长度
      if (currentLength > maxLength) {
        maxLength = currentLength;
      }
    }
  }

  return maxLength;
}
```

## 解法 2：动态规划 / 哈希表边界拼凑法（支持在线处理）

### 思路

- **核心逻辑**：
  我们可以用一个哈希表来动态维护每个数字所在的连续区间的长度。当遍历到一个新数字 `num` 时，如果它之前没出现过，我们就看它的左邻居 `num - 1` 和右邻居 `num + 1` 所在的区间长度。

  > **严格按 DP 五步法展开**：
  1.  **确定 dp 数组（dp table）及下标含义**：`dp` 是一个 `Map<number, number>`。`dp.get(i)` 表示：**如果数字 `i` 是一段连续序列的边界（最左或最右节点），该连续序列的总长度**。
  2.  **确定递推公式（状态转移方程）**：
      当新数字 `x` 进来时：
      - `leftLength = dp.get(x - 1) || 0`
      - `rightLength = dp.get(x + 1) || 0`
      - 当前数字所在的新区间长度：`currentLength = leftLength + rightLength + 1`
      - **关键更新（维护边界不变量）**：更新新区间的左右端点状态，即 `dp.set(x - leftLength, currentLength)` 和 `dp.set(x + rightLength, currentLength)`。由于中间节点不再是边界，它们的值可以不用维护（或直接设为占位符）。
  3.  **dp 数组初始化**：`const dp = new Map<number, number>()`。
  4.  **遍历顺序**：从左到右遍历原数组 `nums`，因为这是处理无序输入。
  5.  **举例推导 dp 数组**：输入 `[4, 2, 3]`：
      - 读入 `4`：左右皆空，长度 `1`。`dp.set(4, 1)`。
      - 读入 `2`：左右皆空，长度 `1`。`dp.set(2, 1)`。
      - 读入 `3`：左邻 `2` (长度1)，右邻 `4` (长度1)。新长度 = `1 + 1 + 1 = 3`。更新边界：左边界 `3-1=2`，右边界 `3+1=4`。所以 `dp.set(2, 3)` 和 `dp.set(4, 3)`。全局最大长度为 `3`。

- **TS 细节**：
  - 使用 `Map` 的好处在于可以干净地处理稀疏键值（不需要像数组一样担心内存越界）。
  - 通过 `dp.has(num)` 来过滤重复元素，避免重复元素引起区间的重复计算和逻辑混乱。

### 代码实现 (LeetCode 提交版)

```ts
/**
 * 复杂度分析：
 * 时间复杂度：O(N) —— 遍历数组一次，每次在 Map 中 get 和 set 的复杂度均摊是 O(1)。
 * 空间复杂度：O(N) —— 最坏情况下所有数字互不相连，Map 中存储了 N 个键值对。
 */

function longestConsecutive2(nums: number[]): number {
  if (nums.length === 0) return 0;

  // dp table：记录所在区间的最大长度。仅区间端点的值保持绝对准确。
  const dp: Map<number, number> = new Map();
  let maxLength: number = 0;

  for (const num of nums) {
    // 已经处理过的数字直接跳过，防止破坏已有的区间边界结构
    if (dp.has(num)) continue;

    // 获取左右邻居所在的区间长度（如果没有，默认为 0）
    const left: number = dp.get(num - 1) ?? 0;
    const right: number = dp.get(num + 1) ?? 0;

    // 状态转移：当前节点连接起左右两个区间
    const currentLength: number = left + right + 1;

    if (currentLength > maxLength) {
      maxLength = currentLength;
    }

    // 核心：只需更新当前数字和整个新区间左右端点的长度值
    // 虽然我们将 num 也存入 dp，但只有端点的值在后续才会被用到
    dp.set(num, currentLength);
    dp.set(num - left, currentLength);
    dp.set(num + right, currentLength);
  }

  return maxLength;
}
```

## 解法 3：原地排序 + 迭代（工程实践中的“反直觉”优化）

### 思路

- **核心逻辑**：
  通常面试要求 $O(N)$，但如果不允许使用额外的 $O(N)$ 空间（如内存严格受限），我们可以直接在原数组上排序。
  排好序后，连续的数字在物理内存上也是连续的。只需遍历一次，如果 `nums[i] === nums[i-1]` 则跳过；如果 `nums[i] === nums[i-1] + 1`，序列长度 `+1`；否则重置为 `1`。
- **TS 细节**：
  - **V8 Timsort 优化**：在 JS/TS 中，对于纯数字数组，`Array.prototype.sort` 的底层通常由引擎优化（如 V8 的 Timsort 或插入/快排变体），虽然时间复杂度是 $O(N \log N)$，但由于没有哈希冲突，没有 `Set/Map` 对象频繁的内存分配与垃圾回收（GC），在 $N \le 10^5$ 的数量级下，排序法的绝对耗时往往比基于 Hash 的方案更短。
  - 必须提供自定义比较函数 `(a, b) => a - b`，因为 TS/JS 的默认 `sort` 会将数字转为字符串按字典序排序，导致负数和多位数出现致命逻辑错误。

### 代码实现 (LeetCode 提交版)

```ts
/**
 * 复杂度分析：
 * 时间复杂度：O(N log N) —— 取决于 V8 引擎底层针对 Array.prototype.sort 的实现。遍历过程只需 O(N)。
 * 空间复杂度：O(1) 或 O(log N) —— 如果允许修改原数组（原地排序），空间由排序算法的递归栈决定，通常为 O(log N)。
 */

function longestConsecutive3(nums: number[]): number {
  if (nums.length === 0) return 0;

  // 原地排序，必须提供数值比较器，否则按字典序排序会出错
  nums.sort((a, b) => a - b);

  let maxLength: number = 1;
  let currentLength: number = 1;

  for (let i = 1; i < nums.length; i++) {
    // 遇到重复元素，忽略，继续下一轮
    if (nums[i] === nums[i - 1]) {
      continue;
    }

    // 遇到连续元素，当前长度 +1
    if (nums[i] === nums[i - 1] + 1) {
      currentLength++;
    } else {
      // 遇到断层，结算并重置
      if (currentLength > maxLength) {
        maxLength = currentLength;
      }
      currentLength = 1;
    }
  }

  // 最后一次序列可能一直延伸到数组结尾，需要在此做一次兜底比较
  return Math.max(maxLength, currentLength);
}
```

## 解法对比与总结

### 1. 适用场景与工程权衡

| 解法                | 时空复杂度                        | 适用场景                                  | 工程权衡与点评                                                                                                                                            |
| :------------------ | :-------------------------------- | :---------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **1. 哈希集合法**   | 时间 $O(N)$<br>空间 $O(N)$        | 面试标准答案、追求极致理论时间复杂度。    | **可读性最佳**。利用 Set 去重且 O(1) 查找。代价是产生较多内存碎片，在极端大数据量时 Hash 碰撞和 GC 开销会导致常数项巨大。                                 |
| **2. DP哈希拼凑法** | 时间 $O(N)$<br>空间 $O(N)$        | **流式数据（Streaming）**、需要在线处理。 | 唯一能在**不一次性拿到所有数据**时，持续维护当前连续区间的算法。代码稍微晦涩，但在某些并查集变种问题上极具扩展性。                                        |
| **3. 原地排序法**   | 时间 $O(N \log N)$<br>空间 $O(1)$ | 空间极其受限的嵌入式/前端端侧场景。       | **空间极致优化**。虽然理论时间不达标，但内存连续性极好，实际执行速度在数据规模 $N \le 10^5$ 时往往比方案 1 还要快。面试时用来作为加分项展示底层理解最佳。 |

### 2. 关键测试样例

以下 5 个测试用例覆盖了所有的代码逻辑分支、边界和异常情况：

1.  **边界用例（空数组）**
    - `输入`：`nums =[]`
    - `期望输出`：`0`
    - `覆盖点`：首行防御性编程，确保对空输入的正确返回。
2.  **边界用例（单元素）**
    - `输入`：`nums = [10]`
    - `期望输出`：`1`
    - `覆盖点`：不会进入 `while` 循环（或解法 3 中的 `for` 循环不产生错误影响），保证最小有效长度正确。
3.  **边界用例（全重复元素）**
    - `输入`：`nums = [2, 2, 2, 2]`
    - `期望输出`：`1`
    - `覆盖点`：防止重复元素导致序列长度计算错误。解法 1 Set 会合并；解法 2 会 `continue` 跳过；解法 3 也会检查 `nums[i] === nums[i-1]` 从而跳过。
4.  **常规用例（跨度分散且乱序）**
    - `输入`：`nums =[100, 4, 200, 1, 3, 2]`
    - `期望输出`：`4` （序列为 `[1, 2, 3, 4]`）
    - `覆盖点`：经典的无序数组测试，检验算法是否能正确忽略不相关的断层（`100`，`200`），独立拼接出最长连续序列。
5.  **极值用例（包含负数、零及跨正负数域）**
    - `输入`：`nums =[0, -1, 1, 2, -2]`
    - `期望输出`：`5` （序列为 `[-2, -1, 0, 1, 2]`）
    - `覆盖点`：检查语言底层的数据结构处理。JS 的 `Array.prototype.sort` 在不传比较函数时会把 `-1` 认为比 `-2` 甚至正数产生错误的字典序，此用例能精准拦截未写 `(a,b)=>a-b` 的错误实现；同时验证零的边界连接。
